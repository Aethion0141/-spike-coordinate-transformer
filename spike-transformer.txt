WHAT IS SPIKE TRANSFORMER?

A geometric AGI that learns from structure, not data. 
Shakespeare understood in 0.93 seconds. No training. No GPUs.

================================================================================
THE THREE CORE SYSTEMS
================================================================================

SYSTEM 1: SPIKE TRANSFORMER (Attention Engine)
------------------------------------------------
Function: Processes attention via geometric distance
Formula:  attention = exp(-distance in 2D space)
Input:    Word positions (X, Y)
Output:   Attention weights between all words
Confidence: attention_confidence (0 to 1)

Key innovation: No Q,K,V matrices. No training. Pure geometry.


SYSTEM 2: SPIKE MEMORY (Storage Engine)
----------------------------------------
Function: Stores spikes with per-axis confidence
Structure: 3-tier memory system
- Tier 1: Short-term (current spike)
- Tier 2: Working memory (last 10 spikes)
- Tier 3: Long-term (consolidated patterns)

Per spike storage: X, Y, conf_x, conf_y, timestamp, access count

Key innovation: Per-axis confidence (knows what it doesn't know)


SYSTEM 3: SPIKE LEARNER (Learning Engine)
------------------------------------------
Function: Learns from triangulation (no labels)
Method:   Original → Reverse → Remix → Pull to center
Input:    Raw sentences (unlabeled)
Output:   Updated word positions
Confidence: learning_confidence (0 to 1)

Key innovation: Self-supervised learning. No teacher. No labels.

================================================================================
SPIKE CLASSIFICATIONS
================================================================================

SPIKE TYPES (By Function)
--------------------------
Type A: SEMANTIC SPIKES - Store word meanings (X, Y coordinates)
Type B: CONFIDENCE SPIKES - Store uncertainty (conf_x, conf_y)
Type C: ATTENTION SPIKES - Store attention patterns (distance matrix)
Type D: MEMORY SPIKES - Store temporal memories (with timestamps)
Type E: LEARNING SPIKES - Store learning patterns (triangulation history)


SPIKE STATES (By Confidence)
-----------------------------
HIGH CONFIDENCE   (conf > 0.7): Trusted knowledge
MEDIUM CONFIDENCE (0.3 < conf < 0.7): Learning phase
LOW CONFIDENCE    (conf < 0.3): Uncertain, needs more data
ZERO CONFIDENCE   (conf = 0): Unknown, system ignores


SPIKE OPERATIONS
----------------
STORE:        Add new spike to memory
RECALL:       Find closest spike by distance
UPDATE:       Move spike position based on learning
MERGE:        Combine spikes with similar meanings
DECAY:        Reduce confidence over time (forgetting)
CONSOLIDATE:  Move short-term to long-term memory

================================================================================
THE SPIKE LEARNING PROCESS
================================================================================

Step 1: INPUT
        Raw sentence: "Caesar was a great general"

Step 2: SPIKE ENCODING
        Each word becomes a spike with random (X, Y) and conf_x=0.5, conf_y=0.5

Step 3: TRIANGULATION
        Create Original, Reverse, Remix versions
        Encode all three to (x, y) positions

Step 4: CENTER CALCULATION
        center = (orig + rev + remix) / 3

Step 5: SPIKE UPDATE
        Move each word's spike toward center
        Update confidence based on movement

Step 6: MEMORY STORAGE
        Store updated spikes in 3-tier memory

Step 7: ATTENTION COMPUTATION
        distance = √[(Δx·conf_x)² + (Δy·conf_y)²]
        attention = exp(-distance)

Step 8: OUTPUT
        Understanding emerges. Relationships visible. Themes extracted.

================================================================================
THE FORMULAS
================================================================================

ATTENTION FORMULA
-----------------
attention = exp(-distance)

Where:
distance = √[(Δx·conf_x)² + (Δy·conf_y)²]
Δx = spike1.x - spike2.x
Δy = spike1.y - spike2.y
conf_x = spike1.conf_x × spike2.conf_x
conf_y = spike1.conf_y × spike2.conf_y


LEARNING RULE
-------------
new_position = old_position + (center - old_position) × learning_rate × avg_conf


CONFIDENCE UPDATE RULE
----------------------
if accessed correctly:   confidence += 0.1
if accessed incorrectly: confidence -= 0.15
if not accessed:         confidence decays over time
